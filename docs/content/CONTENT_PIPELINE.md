# Paper content pipeline

1. Confirm permission or a defensible licence before publishing a paper.
2. Save immutable exam and solution PDFs with school, year, course and version metadata.
3. Record question and subpart IDs, maximum marks, exam page, solution page and NESA syllabus area.
4. Verify page mappings visually and check that all subpart marks total the paper maximum.
5. Have a second reviewer compare every mapping with the PDF.
6. Release through a pull request with automated checks and a completed manual test sheet.

The current flashcards show full source pages. A future crop pipeline may store bounding boxes for individual parts, but must preserve surrounding diagrams, shared stems and copyright attribution.
