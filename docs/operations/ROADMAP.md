# Roadmap and ownership

## Phase 1 — dependable prototype

Finish the full attempt loop, add browser regression tests, validate all Abbotsleigh mappings and complete student usability sessions.

## Phase 2 — content system

Create a reviewed paper manifest format, ingestion tools, rights register and an admin quality-control screen. Add a small, high-quality set across Advanced and Extension courses.

## Phase 3 — accounts and pilots

Add secure persistence, privacy controls, cross-device history and tutor/school pilots. Instrument retention and learning-loop completion.

## Phase 4 — sustainable launch

Test pricing, support, incident response, backups, accessibility review and scalable content operations.

Every feature has one owner, reviewer, acceptance criteria and target release. Releases require passing automated tests, the manual checklist, content approval and a rollback note.
